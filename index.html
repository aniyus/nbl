What NBL does

NBL (let's say "nibble") is a small JavaScript library that allows you to dynamically load your JavaScript files in parallel with the other content of your site. Traditionally including your scripts in the <head> tag of your pages would cause a browser to load the files in one after another, in some cases even blocking other files from loading.

However, dynamically loading your scripts has one drawback: in some cases the scripts are loaded out of order. If you want to load jQuery and a jQuery plugin, dynamically loading could mean the plugin initialises before jQuery even finishes loading. NBL solves this by letting you define your main script separately from your plugins, in effect your plugins load after the main script has finished loading.

How NBL works

Simply include the following tag in your page:

<script type="text/javascript" src="nbl.js" opt="{ urchin: 'http://www.google-analytics.com/urchin.js', plugins: [ 'jquery.lightbox.min.js', 'jquery.carousel.min.js' ], ready: my_ready_function }"></script>

This will do the following:

  1. It will load the latest version of jQuery from http://ajax.googleapis.com/ajax/libs/jquery/1/jquery.min.js.
  2. It will load the Urchin script from Google Analytics.
  3. After jQuery has loaded, it will start loading the jQuery plugins defined in the plugins array above.
  4. When jQuery has loaded, it will attach the my_ready_function() to jQuery's document.ready() method. Sometimes your pages are rendered before jQuery is fully initialised, in those cases NBL will wait until all scripts have loaded and call the my_ready_function() itself.
After NBL has done its job you can verify a few things through the global nbl object (you can change the name of this object by specifying name: in the options). Read more about the options in the documentation.

NB. While jQuery loads by default, it is possible to replace it with your favourite library by specifying the jquery: option with your script. Any plugins specified will then load after this script has loaded.
